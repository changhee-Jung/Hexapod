Model
1. 헥사포드 파라메터
 1.1. Base 
  1.1.1. 설정(설계) 좌표
    - Base 판의 연결된 조인트의 위치: B1 ~ B6
    - Base 판의 반지름
    - 두 힌지 사이의 Offset 각도
   => 각 조인트 포인트를 계산하기 위해 필요(만약 설계 데이터로 존재할 경우, 그 값을 가져옴)
   1.1.2. 상태 값
    - 중심 점의 현 위치, 회전 값(x, y, z, r, p, y) 
 1.2. Upper  
   1.2.1. 설정(설계) 좌표
    - Upper 판의 연결된 조인트의 위치: P1 ~ P6
    - Upper 판의 반지름
    - 두 힌지 사이의 Offset 각도
   => 각 조인트 포인트를 계산하기 위해 필요(만약 설계 데이터로 존재할 경우, 그 값을 가져옴)
    1.2.2. 상태 값
    - 중심 점의 현 위치, 회전 값(x, y, z, r, p, y)
    1.2.3. 목표 값
    - 목표 지점의 위치, 회전 값(x ,y, z, r, p, y)
    - 목표 지점 기반의 예상 경로

2. 각 엑추에이터 파라메터
    - 엑추에이터 번호
    - 엑추에이터 상태
    - 엑추에이터 현 위치
    - 엑추에이터 최소, 최대 이동 범위
    - 엑추에이터 계수(최소, 최고 속도 파라메터)

2. 메소드
 2.1. 좌표 변환
    - 한 포인트를 기준으로 120도 씩 변환하여 각 조인트 위치를 설정하는 메소드(시작점, 반지름) -> 각 판에 대한 매트릭스 생성
    - 회전 변환 행렬 계산 매트릭스(오일러 각 또는 브라이언트 각)
    - 위치 변환 행렬 계산 매트릭스
    - Norm 계산(절대값(1), 또는 sqrt 기법(2) 이용)

 2.2. 경로 계획 관련
    - 갱신 주기 어떻게 할지 필요
    - 이동 완료 되었는지 상태 확인  
   
 2.3 엑추에이터 구성
    -> 현재 엑추에이터 상태(위치, 상태) 획득을 기준으로 작성.
    - 가상 테스트
    - 모터

View
 1. 입력
   - 설계 자료(Base, Upper 등의 좌표 값) 설정할 수 있도록 구현
   - EndPoint의 위치, 회전 각도 - > (x, y, z, r, p, y)
 
 2. 출력
  2.1 목표 위치 계산
    - 좌표계 변환 기반의 최종 엑추에이터 길이 값 표시(L1 ~ L6)
    - 각 엑추에이터의 최종 위치 값을 기반으로 현 위치와의 차이로 모션 또는 경로 그래프 생성
    - 각 엑추에이터의 상태 표시(위치, State)
    - EndPoint 이동 예상 경로 표시

  3. 에러 표시
    - 상태 조건에 따른 에러 표시
    - 엑추에이터 에러 발생에 대한 표시
     
Controller
 1. 메인
  1.1.시작
    1.1.1. 버튼 관련
    - 좌표계 계산 시작 버튼
    - 모션 또는 경로 그래프 생성 시작 버튼(이전 것이 그려져 있어도 새로 갱신)
    - 실 엑추에이터 이동 시작 버튼
  1.2. 정지
   1.2.1. 버튼 관련
    - 엑추에이터 이동 정지 버튼
  1.1.3. 초기화
    1.3.1. 버튼 관련
    - 모션 또는 경로 그래프 초기화 
 
2. 엑추에이터
 2.1. 시작/ 해제
   - 엑추에이터 On/Off 설정 
 2.2. 이동 관련
    - 각 엑추에이터 이동 제어(위치, 속도 설정)
    - 각 엑추에이터 튜닝(PID 설정 등)
